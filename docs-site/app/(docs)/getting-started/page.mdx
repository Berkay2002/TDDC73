# Getting Started

Get up and running with Primitive UI in your Flutter project in just a few minutes.

## What is Primitive UI?

Primitive UI is a Flutter GUI library built entirely from scratch using only primitive components:

- **CustomPaint** and **Canvas** for rendering
- **GestureDetector** for user input
- **Custom RenderBox** for layout

No high-level widgets like `Card`, `Column`, `Switch`, or `Stack` are used in the implementation. This makes Primitive UI an excellent educational tool for understanding how Flutter's rendering engine works under the hood.

## Installation

<Steps>

### Add Primitive UI to Your Project

Since Primitive UI is currently a local package, add it to your `pubspec.yaml`:

```yaml filename="pubspec.yaml" {3-4}
dependencies:
  flutter:
    sdk: flutter
  primitive_ui:
    path: ../primitive_ui
```

<Callout type="warning">
**Important:** Adjust the `path` to match your project structure. The path is relative to your project's root directory.
</Callout>

### Install Dependencies

Run the following command in your terminal:

```bash
flutter pub get
```

You should see output like:

```
Running "flutter pub get" in your_project...
Resolving dependencies...
+ primitive_ui 0.0.1 from path ../primitive_ui
Changed 1 dependency!
```

### Import the Library

In your Dart files, import Primitive UI:

```dart filename="lib/main.dart"
import 'package:primitive_ui/primitive_ui.dart';
```

### Verify Installation

Create a simple test to verify everything works:

```dart filename="lib/main.dart"
import 'package:flutter/material.dart';
import 'package:primitive_ui/primitive_ui.dart';

void main() {
  runApp(MyApp());
}

class MyApp extends StatelessWidget {
  @override
  Widget build(BuildContext context) {
    return MaterialApp(
      home: Scaffold(
        body: Center(
          child: CustomCard(
            child: Text('Primitive UI Works!'),
          ),
        ),
      ),
    );
  }
}
```

Run your app:

```bash
flutter run
```

You should see a white card with "Primitive UI Works!" in the center.

</Steps>

**Step 4:** Build the UI using Primitive UI components

```dart
@override
Widget build(BuildContext context) {
  return Scaffold(
    appBar: AppBar(title: Text('Settings')),
    body: Padding(
      padding: EdgeInsets.all(16.0),
      child: VStack(
        spacing: 16.0,
        children: [
          _buildSettingItem('Notifications', _notificationsEnabled, (v) {
            setState(() => _notificationsEnabled = v);
          }),
          _buildSettingItem('Dark Mode', _darkModeEnabled, (v) {
            setState(() => _darkModeEnabled = v);
          }),
        ],
      ),
    ),
  );
}

Widget _buildSettingItem(
  String label,
  bool value,
  ValueChanged<bool> onChanged,
) {
  return CustomCard(
    elevation: 2.0,
    child: Row(
      mainAxisAlignment: MainAxisAlignment.spaceBetween,
      children: [
        Text(label),
        CustomToggleSwitch(
          value: value,
          onChanged: onChanged,
        ),
      ],
    ),
  );
}
```

## Quick Reference

### Common Patterns

**Creating a card:**
```dart
CustomCard(
  child: Text('Content'),
)
```

**Vertical layout:**
```dart
VStack(
  spacing: 16.0,
  children: [
    Text('First'),
    Text('Second'),
  ],
)
```

**Layering widgets:**
```dart
ZStack(
  alignment: Alignment.center,
  children: [
    Container(width: 200, height: 200, color: Colors.blue),
    Icon(Icons.star, size: 50),
  ],
)
```

**Toggle switch:**
```dart
CustomToggleSwitch(
  value: isEnabled,
  onChanged: (value) => setState(() => isEnabled = value),
)
```

## Troubleshooting

### Package Not Found

If you see `Error: Cannot find package 'primitive_ui'`:

1. Verify the path in `pubspec.yaml` is correct
2. Run `flutter pub get`
3. Restart your IDE

### Import Errors

If you see "Undefined class" errors:

1. Ensure you've imported: `import 'package:primitive_ui/primitive_ui.dart';`
2. Run `flutter pub get`
3. Restart the Dart Analysis Server

### Shadow Not Rendering

If `CustomCard` shadows don't appear:

1. Add padding around the card to give space for the shadow
2. Ensure the card isn't clipped by its parent
3. Try increasing the `elevation` value

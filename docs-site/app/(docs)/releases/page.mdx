<div className="flex items-start gap-4 p-6 rounded-lg bg-teal-50 dark:bg-teal-950/30 border border-teal-200 dark:border-teal-800 my-8">
  <span className="text-2xl shrink-0">âœ¨</span>
  <div>
    <h4 className="text-base font-semibold !text-white dark:text-white mb-1">Interactive Primitives: Button & Input</h4>
    <p className="text-sm !text-white dark:text-white mb-2">Introduced `PrimitiveButton` and `PrimitiveInput`, establishing the foundation for interactive forms and actions with a focus on composability and lower-level control.</p>

    - Multiple **variants** (primary, secondary, destructive, outline, ghost, link).
    - Built-in handling for **loading states** and **disabled states**.
- **New Component**: `PrimitiveInput` introduces a composable text input.
    - Built directly on `EditableText` to avoid Material bloat.
    - Supports **visual variants** (outline, filled, flushed) and decoration slots.

## November 24, 2025 (v0.0.5)

### Version 0.0.5 

<div className="flex items-start gap-4 p-6 rounded-lg bg-purple-50 dark:bg-purple-950/30 border border-purple-200 dark:border-purple-800 my-8">
  <span className="text-2xl shrink-0"></span>
  <div>
    <h4 className="text-base font-semibold !text-white dark:text-white mb-1">Layout Intelligence & Implicit Animations</h4>
    <p className="text-sm !text-white dark:text-white mb-2">Introduced standard alignment APIs and smooth implicit animations for enhanced layout control and visual polish.</p>
    <code className="text-xs bg-purple-900 text-white dark:bg-purple-900/50 dark:text-white px-2 py-1 rounded">CrossAxisAlignment / Animated Styles / TweenAnimationBuilder</code>
  </div>
</div>

- **Layout Intelligence**: `VStack` and `HStack` now support standard Flutter `CrossAxisAlignment` (start, center, end, stretch), replacing custom enums.
- **Implicit Animations**: `CustomCard` now smoothly animates changes to color, elevation, and shadow.
- **Implicit Animations**: `CustomSlider` now animates programmatic value changes while keeping drag interactions immediate.
- **API Standardization**: Standardized alignment properties across layout components.

### Version 0.0.4

  <span className="text-2xl shrink-0"></span>

- **CustomCard**: Added semantic button/container roles and `semanticsLabel` parameter.
- **CustomToggleSwitch**: Added semantic switch role, toggled state, and `semanticsLabel`.

## November 23, 2025 (v0.0.3)

### Version 0.0.3

<div className="flex items-start gap-4 p-6 rounded-lg bg-indigo-50 dark:bg-indigo-950/30 border border-indigo-200 dark:border-indigo-800 my-8">
  <span className="text-2xl shrink-0"></span>
  <div>
    <h4 className="text-base font-semibold !text-white dark:text-white mb-1">Feature Enhancements & Layout Updates</h4>
    <p className="text-sm !text-white dark:text-white mb-2">Significant updates across several components, introducing flexible layouts, absolute positioning, improved interaction, and new progress indicator capabilities.</p>
    <code className="text-xs bg-indigo-900 text-white dark:bg-indigo-900/50 dark:text-white px-2 py-1 rounded">CustomFlexible / CustomPositioned / DeterminateProgress</code>
  </div>
</div>

- **CustomCard**: Added `shadowColor` and `onTap` (with visual feedback) support.
- **HStack**: Introduced `HCustomFlexible` and `HCustomExpanded` for proportional sizing (flex layout).
- **VStack**: Further enhanced with `CustomFlexible` and `CustomExpanded` for robust flex layout.
- **ZStack**: Added `CustomPositioned` support for absolute positioning within the stack.
- **CustomSlider**: Enhanced with `onChangeStart`, `onChangeEnd` callbacks, and `thumbColor` customization.
- **CustomCircularProgress**: Now supports determinate progress (pass a `value` between 0.0 and 1.0) alongside indeterminate mode.

## November 23, 2025 (v0.0.2)

### Version 0.0.2

<div className="flex items-start gap-4 p-6 rounded-lg bg-blue-50 dark:bg-blue-950/30 border border-blue-200 dark:border-blue-800 my-8">
  <span className="text-2xl shrink-0"></span>
  <div>
    <h4 className="text-base font-semibold !text-white dark:text-white mb-1">Horizontal Layouts & Interactive Widgets</h4>
    <p className="text-sm !text-white dark:text-white mb-2">Introduced horizontal stacking with alignment control and new interactive components for value selection and loading states.</p>
    <code className="text-xs bg-blue-900 text-white dark:bg-blue-900/50 dark:text-white px-2 py-1 rounded">HStack / CustomSlider / CustomCircularProgress</code>
  </div>
</div>

- Added `HStack` layout: A horizontal stack layout with alignment options
- Added `MainAxisAlignment` support to `VStack` and `HStack` for distributing free space (start, center, end, spaceBetween, spaceAround, spaceEvenly)
- Improved layout logic for stacks to correctly handle single children with alignment
- Added `CustomSlider` component: A slider built with `CustomPaint` and `GestureDetector`
- Added `CustomCircularProgress` component: An indeterminate progress indicator built with `CustomPaint` and `AnimationController`

## November 15, 2025

### Version 0.0.1

Initial release of Primitive UI.

<div className="flex items-start gap-4 p-6 rounded-lg bg-amber-50 dark:bg-amber-950/30 border border-amber-200 dark:border-amber-800 my-8">
  <span className="text-2xl shrink-0"></span>
  <div>
    <h4 className="text-base font-semibold !text-white dark:text-white mb-1">Core Primitives & Foundation</h4>
    <p className="text-sm !text-white dark:text-white mb-2">The initial release establishing the low-level rendering architecture and core layout components.</p>
    <code className="text-xs bg-amber-900 text-white dark:bg-amber-900/50 dark:text-white px-2 py-1 rounded">CustomPaint / RenderBox / VStack / ZStack</code>
  </div>
</div>

- Added `CustomCard` component: A container with shadow, rounded corners, and padding
- Added `CustomToggleSwitch` component: An animated toggle switch
- Added `VStack` layout: A vertical stack layout with alignment options
- Added `ZStack` layout: A layered stack layout with alignment options
- Implemented using only low-level primitives: `CustomPaint`, `Canvas`, `GestureDetector`, and custom `RenderBox`
